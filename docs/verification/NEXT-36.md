# NEXT-36

## 작업 이름
현재 작업트리 실제 코드 변경 고정 및 검증

## 목적
현재 main 작업트리에 남아 있는 modified / untracked 파일을 누락 없이 정리하고,
실제 반영 대상과 제외 대상을 명확히 분리한 뒤,
반영 대상에 대해 compile / smoke 테스트를 수행하고 커밋 가능한 상태로 만든다.

## 이번 단계의 핵심 원칙
- git status에 잡힌 파일을 하나도 빠뜨리지 않는다
- 각 파일은 반영 / 삭제 / 제외 중 하나로 반드시 판정한다
- 코드 파일은 반드시 테스트한다
- 문서만 커밋하고 코드가 빠지는 상황을 금지한다

## 이번 단계의 성공 기준
- modified / untracked 파일 전체 목록이 WORK-36에 기록됨
- 각 파일의 처리 방침이 명시됨
- 반영 대상 코드에 대한 compile / smoke 테스트가 성공함
- 실제 add/commit/push가 수행됨
